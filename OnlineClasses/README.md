# Online Classes Platform

Distributed system developed for the **Network Programming** course at **ORT University (Uruguay)**.  
The project implements a platform that allows users to create, manage, and participate in online classes using multiple networking technologies and distributed system concepts.

## Overview

The platform enables users to:

- Create and manage online classes
- Enroll in available classes
- Search and filter classes
- Participate in live chat during classes
- View their class participation history

The system is composed of several independent services that communicate using different networking technologies.

## System Components

### Main Server

Responsible for the core platform logic, including:

- User authentication
- Class creation and management
- Enrollment management
- Class scheduling
- Webhook notifications before classes start

Technologies used:

- TCP Sockets  
- Custom application protocol  
- Async/Await with Tasks  
- gRPC communication with other services  
- RabbitMQ for messaging  
- Docker deployment  

---

### Client Application

Allows users to interact with the system.

Users can:

- Register and authenticate
- Create and manage classes
- Search and filter available classes
- Enroll or cancel enrollment
- View class history

Technologies used:

- TCP sockets
- Custom communication protocol
- Async/Await networking

---

### Chat Server

Handles live chat sessions for online classes and manages the communication between participants.

Technologies used:

- WebSockets for real-time communication
- Async networking

---

### Chat Client

Used by participants to communicate during live classes.

Technologies used:

- WebSockets
- Real-time message handling

---

### Logging Server

Responsible for collecting and exposing logs generated by the system.

Logs can be filtered using different criteria and accessed remotely.

Technologies used:

- RabbitMQ for receiving log messages
- REST API for remote log access
- Docker deployment

---

## Communication Protocol

The system implements a **custom TCP protocol** for communication between the client and the main server.

Message structure:

HEADER | CMD | LENGTH | DATA

Field description:

- **HEADER** – Message type (request or response)
- **CMD** – Command identifier
- **LENGTH** – Size of the payload
- **DATA** – Message content

Messages are transmitted using **network streams**.
